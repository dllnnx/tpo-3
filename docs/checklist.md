## По требованиям лабораторной работы

- [x] Тестовое покрытие сформировано на основе набора прецедентов использования (12 UC).
- [x] Тестирование автоматизировано с помощью Selenium.
- [x] Тесты исполняются в Chrome и Firefox.
- [x] Шаблоны тестов соответствуют идее Selenium IDE, реализация — Selenium WebDriver.
- [x] Поиск элементов осуществляется через **XPath** (без `By.id`/`By.cssSelector`).
- [x] Динамически генерируемые элементы (React/Vue с обфусцированными классами) находятся через `data-ti`-атрибуты + label-based XPath.
- [x] Тесты можно запускать в обоих браузерах параллельно (`testng-parallel.xml`) и в каждом отдельно (`testng-chrome.xml`, `testng-firefox.xml`).
- [x] Использованы только явные ожидания `WebDriverWait` (никаких `Thread.sleep`).
- [x] Применён паттерн **Page Object** (14 классов в `pages/`).
- [x] В отчёте присутствуют:
  - [x] Use-case диаграмма (`docs/diagram.puml`).
  - [x] Чек-лист (этот файл).
  - [x] Описание тестовых сценариев (`docs/scenarios.md`).
  - [x] Описание прецедентов использования (`docs/use-cases.md`).

## Что в тестах НЕ проверяется (out of scope)

- Бронирование и оплата билетов (требует авторизации и реальных денежных средств).
- Личный кабинет и история заказов (требует авторизации).
- Мобильная вёрстка — тесты прогоняются на десктопном размере 1440×900.
- Юзабилити и визуальная регрессия — Selenium тестирует только функциональность.
- Производительность — нет нагрузочных тестов.
