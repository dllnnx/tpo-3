## UC-01: Поиск ж/д билетов с указанием маршрута и даты

**Актор:** Гость

**Предусловия:**
- У пользователя есть рабочий браузер.
- Сайт `tutu.ru` доступен.

**Основной поток:**
1. Гость открывает главную страницу `https://www.tutu.ru/`.
2. Гость переходит в раздел «Ж/д билеты» (`https://www.tutu.ru/poezda/`).
3. Гость кликает в поле «Откуда» и вводит начало названия города отправления.
4. Гость дожидается появления выпадающего списка-саджеста и выбирает нужный город.
5. Гость кликает в поле «Куда», вводит начало названия города назначения и выбирает его в саджесте.
6. Гость кликает в поле «Дата» — открывается календарь jQuery-UI.
7. Гость выбирает желаемую дату поездки (например, +30 дней от сегодня).
8. Гость нажимает кнопку «Узнать расписание и цены».

**Постусловие (успех):**
- URL имеет вид `/poezda/{From}/{To}/?date=DD.MM.YYYY`.
- На странице отображается список карточек поездов с временем отправления, прибытия, длительностью пути и типами вагонов.

**Альтернативные сценарии:**
- Если пользователь не выбрал саджест: скрытые поля идентификаторов городов остаются пустыми, при submit система остаётся на форме поиска.

## UC-02: Применение фильтра «Сапсан» к списку поездов

**Актор:** Гость

**Предусловия:**
- Выполнен UC-01: пользователь находится на странице с результатами поиска ж/д билетов, в списке есть как минимум один Сапсан.

**Основной поток:**
1. Гость находит в области фильтров чип «Сапсан».
2. Гость кликает на чип «Сапсан».
3. Список результатов перерисовывается.

**Постусловие:**
- Чип «Сапсан» получает CSS-класс активного состояния (`selected_*`).
- В списке остаются только карточки поездов, в названии которых упоминается «Сапсан».
- Минимальная цена в шапке результатов изменяется.

## UC-03: Переключение даты в date-strip результатов ж/д

**Актор:** Гость

**Предусловия:**
- Выполнен UC-01.

**Основной поток:**
1. На странице результатов гость видит date-strip с соседними датами и ценами.
2. Гость кликает на одну из соседних дат.
3. Страница обновляется с новой датой в URL.

**Постусловие:**
- Параметр `date=` в URL изменился.
- Список карточек поездов перерисован для новой даты, остаётся не менее одной карточки.

## UC-04: Поиск авиабилетов с заполнением формы и календаря

**Актор:** Гость

**Предусловия:**
- Сайт `avia.tutu.ru` доступен.

**Основной поток:**
1. Гость открывает `https://avia.tutu.ru/`.
2. Гость закрывает баннер согласия с cookies (если показан).
3. Гость кликает в поле «Откуда» — на десктопе открывается выпадающий контейнер `dropdown-suggest-container`.
4. Гость вводит «Москва» и выбирает первую подсказку из списка `dropdown-item`.
5. Гость кликает в поле «Куда», вводит «Санкт-Петербург», выбирает подсказку.
6. Гость кликает в поле «Когда» — открывается календарь авиа.
7. Гость выбирает дату вылета (например, +14 дней) и нажимает «Выбрать».
8. Гость нажимает кнопку «Найти авиабилеты».

**Постусловие:**
- URL имеет вид `https://avia.tutu.ru/f/Moskva/Sankt-Peterburg/?...`.
- На странице отображаются карточки рейсов с авиакомпанией, временем, длительностью, ценой.

**Альтернативный поток:**
- Если форма заполнена частично — submit вызывает подсветку незаполненных полей красной рамкой (`error_*` класс).

## UC-05: Применение фильтра «С багажом» к списку рейсов

**Актор:** Гость

**Предусловия:**
- Гость находится на странице результатов поиска авиабилетов (`avia.tutu.ru/f/...`).

**Основной поток:**
1. Гость находит чекбокс-фильтр «С багажом» в форме над результатами.
2. Гость кликает по чекбоксу.

**Постусловие:**
- Состояние чекбокса (`checked`) меняется.
- В списке карточек начинают преобладать варианты с включённым багажом, в карточках появляется упоминание «Багаж» / «с багажом».

## UC-06: Поиск автобусных рейсов

**Актор:** Гость

**Предусловия:**
- Сайт `bus.tutu.ru` доступен.

**Основной поток:**
1. Гость открывает `https://bus.tutu.ru/`.
2. Гость кликает в поле с placeholder «Откуда», вводит «Москва» и выбирает подсказку из выпадающего списка (`styles__dropdown_item`).
3. Гость кликает в поле «Куда», вводит «Тула», выбирает подсказку.
4. Гость кликает в поле «Дата» и выбирает «Завтра».
5. Гость нажимает «Найти билеты».

**Постусловие:**
- Происходит переход на страницу с маршрутом (URL содержит транслитерированные названия городов и параметр даты).
- На странице видны автобусные рейсы с временем отправления, перевозчиком и ценой, либо текст «Москва»/«Тула» в заголовке маршрута.

## UC-07: Поиск отелей

**Актор:** Гость

**Предусловия:**
- Сайт `hotel.tutu.ru` доступен.

**Основной поток:**
1. Гость открывает `https://hotel.tutu.ru/`.
2. Гость кликает в поле направления (`data-ti=hotels-destination-input`), вводит «Сочи», выбирает подсказку.
3. Гость кликает в поле дат (`data-ti=date-input`) и выбирает заезд (+30 дней) и выезд (+32 дня) в открывшемся календаре.
4. Гость нажимает «Найти».

**Постусловие:**
- Страница результатов на `hotel.tutu.ru` отображает карточки отелей с названием, рейтингом и ценой за ночь, либо в заголовке упоминается «Сочи».

## UC-08: Просмотр расписания электричек

**Актор:** Гость

**Предусловия:**
- Сайт `tutu.ru/prigorod/` доступен.

**Основной поток:**
1. Гость открывает `https://www.tutu.ru/prigorod/`.
2. Форма по умолчанию предзаполнена маршрутом (например, Москва → Мытищи). При необходимости гость заполняет поля «Откуда» и «Куда».
3. Гость нажимает «Показать расписание».

**Постусловие:**
- URL изменяется (содержит `/prigorod/` и сегменты станций).
- На странице видно расписание (несколько строк с временем формата `HH:MM`) либо упоминание «Расписание» / «электричка».

## UC-10: Открытие модалки авторизации и ввод email

**Актор:** Гость

**Предусловия:**
- Сайт `www.tutu.ru` доступен.

**Основной поток:**
1. Гость открывает главную `https://www.tutu.ru/`.
2. Гость нажимает кнопку «Войти» в правом верхнем углу.
3. Открывается модалка/форма авторизации.

**Постусловие:**

## UC-11: Поиск в Справочной

**Актор:** Гость

**Предусловия:**
- Раздел Справочной доступен на `tutu.ru/2read/`.

**Основной поток:**
1. Гость открывает `https://www.tutu.ru/2read/`.
2. Гость вводит запрос в поле поиска (например, «возврат билета»).

**Постусловие:**

- На странице видны заголовки (h1/h2) с релевантным контентом.

## UC-12: Попытка поиска авиабилетов с одинаковыми городами (негативный сценарий)

**Актор:** Гость

**Предусловия:**
- Сайт `avia.tutu.ru` доступен.

**Основной поток:**
1. Гость открывает `https://avia.tutu.ru/`.
2. Гость заполняет поле «Откуда» городом «Москва».
3. Гость заполняет поле «Куда» городом «Москва» (так же).
4. Гость нажимает «Найти авиабилеты».

**Постусловие (ожидаемое):**
- Переход на страницу результатов `/f/...` **не происходит** — URL остаётся `https://avia.tutu.ru/`.
- Либо появляется валидационное сообщение об ошибке, либо незаполненные/некорректные поля подсвечиваются красной рамкой.

**Цель сценария:** убедиться, что система валидирует невозможный маршрут и не отправляет пользователя на пустую страницу результатов.
